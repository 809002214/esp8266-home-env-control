- `HTTP_OTA_NO_UPDATE`
- `HTTP_OTA_FAIL_xxx`

## system/reset

这个主题是命令主题。

支持命令：

- `RESTART`
- `ARM_RESET`
- `RESET:CONFIRM`

区别：

- `RESTART`：只重启设备，不清除配置
- `ARM_RESET`：打开 60 秒恢复出厂确认窗口
- `RESET:CONFIRM`：仅在窗口内生效，执行恢复出厂

注意：

- 远程恢复出厂默认禁用（配网页字段 `允许远程恢复出厂` 默认是 `0`）

## ota/set

这个主题用于远程触发 OTA。

支持命令：

- `UPDATE`
- `UPDATE:/firmware/你的固件名.bin`
- `STATE`

区别：

- `UPDATE`：按当前配网页里保存的 `OTA下载端口` 和 `OTA固件路径` 执行升级
- `UPDATE:/firmware/xxx.bin`：临时用指定路径升级，不改已保存配置
- `STATE`：让设备立即重发一次当前 OTA 状态

## ota/state

这个主题是保留消息，用来告诉页面和后台当前 OTA 处于什么状态。

典型内容：

```json
{
  "device_id": "home-room1-env01-ABC123",
  "device_name": "EnvMonitor-ABC123",
  "firmware_version": "1.3.0",
  "ota_in_progress": false,
  "arduino_ota_ready": true,
  "ota_state": "ARDUINO_OTA_READY",
  "ota_port": 1880,
  "ota_path": "/firmware/esp8266-home-env-control.bin"
}
```

## 故障恢复策略

当前代码的恢复逻辑是：

- `Wi-Fi` 断开：每隔 10 秒自动重连；持续 5 分钟还不行就进入恢复配网页
- `MQTT` 断开：每隔 5 秒自动重连；持续 5 分钟还不行就进入恢复配网页
- `NTP` 未同步：不重启，不算致命错误；连网后自动重复发起校时
- `DS18B20` 读取失败：标记 `ds18b20_ok=false`，继续下一轮自动重试
- `DHT11` 读取失败：标记 `dht11_ok=false`，继续下一轮自动重试
- `OLED` 初始化失败：每隔 30 秒重新尝试初始化
- 配置错误：阻止进入正常运行，停留在配网修正流程
