# 系统主题与状态字段

这一版固件除了原来的温湿度和开关主题，还新增了更适合“产品化”的系统主题。

## 主题列表

- `home/room1/env01/status`
- `home/room1/env01/temperature`
- `home/room1/env01/humidity`
- `home/room1/env01/switch/state`
- `home/room1/env01/switch/set`
- `home/room1/env01/device/info`
- `home/room1/env01/system/state`
- `home/room1/env01/system/event`
- `home/room1/env01/system/reset`

## device/info

这个主题是保留消息，用来描述“这台设备是谁”。

典型内容：

```json
{
  "device_id": "home-room1-env01-ABC123",
  "device_name": "EnvMonitor-ABC123",
  "serial_number": "SN-ABC123",
  "model": "ESP8266_ENV_MONITOR",
  "hardware_revision": "NODEMCU_V3",
  "firmware_version": "1.2.0",
  "mac": "AA:BB:CC:DD:EE:FF"
}
```

## system/state

这个主题也是保留消息，用来描述“这台设备现在状态怎么样”。

主要字段：

- `config_valid`
- `wifi_connected`
- `mqtt_connected`
- `oled_ready`
- `ds18b20_ok`
- `dht11_ok`
- `time_synced`
- `local_time`
- `timezone_hours`
- `temperature_offset`
- `humidity_offset`
- `ip`
- `wifi_ssid`
- `wifi_rssi`
- `uptime_sec`
- `reset_reason`
- `last_error`
- `config_error`

## system/event

这个主题用于记录一些短事件，比如：

- `MQTT_CONNECTED`
- `NTP_TIME_SYNCED`
- `SWITCH_ON`
- `SWITCH_OFF`
- `REMOTE_RESTART`
- `REMOTE_FACTORY_RESET`

## system/reset

这个主题是命令主题。

支持两种命令：

- `RESTART`
- `RESET`

区别：

- `RESTART`：只重启设备，不清除配置
- `RESET`：恢复出厂，清除 Wi-Fi 和 MQTT 配置，然后重启

## 故障恢复策略

当前代码的恢复逻辑是：

- `Wi-Fi` 断开：每隔 10 秒自动重连；持续 5 分钟还不行就进入恢复配网页
- `MQTT` 断开：每隔 5 秒自动重连；持续 5 分钟还不行就进入恢复配网页
- `NTP` 未同步：不重启，不算致命错误；连网后自动重复发起校时
- `DS18B20` 读取失败：标记 `ds18b20_ok=false`，继续下一轮自动重试
- `DHT11` 读取失败：标记 `dht11_ok=false`，继续下一轮自动重试
- `OLED` 初始化失败：每隔 30 秒重新尝试初始化
- 配置错误：阻止进入正常运行，停留在配网修正流程
