# 烧录与联调

这份说明按“第一次就尽量跑起来”的思路来写。

## 1. 安装 Arduino IDE 的 ESP8266 支持

在 Arduino IDE 里：

1. 打开 `File -> Preferences`
2. 在附加开发板管理器网址里加入：

```text
http://arduino.esp8266.com/stable/package_esp8266com_index.json
```

3. 打开 `Tools -> Board -> Boards Manager`
4. 搜索 `esp8266`
5. 安装 `ESP8266 by ESP8266 Community`

## 2. 安装依赖库

打开 `Sketch -> Include Library -> Manage Libraries`，把下面文件里列的库都装上：

- [libraries.txt](../arduino/home_env_monitor/libraries.txt)

## 3. 打开示例工程

打开：

- [home_env_monitor.ino](../arduino/home_env_monitor/home_env_monitor.ino)

然后你只需要确认文件顶部这些基础配置：

- `DEVICE_ID`
- `DEVICE_MODEL`
- `FIRMWARE_VERSION`
- `TOPIC_*`
- `CONFIG_AP_PASSWORD`

这版代码已经改成“手机网页配网”，所以不需要再把家里 Wi-Fi 名称和密码写死在代码里。

## 4. 推荐开发板参数

先按下面这组常见参数：

- Board: `NodeMCU 1.0 (ESP-12E Module)`
- Upload Speed: `115200`
- CPU Frequency: `80 MHz`
- Flash Size: `4MB`
- Port: 选择你的 `CH340` 串口

如果串口没有出现，先安装 `CH340` 驱动。

## 5. 第一次上电建议顺序

如果你还不熟，别一上来把所有模块全接上。

按这个顺序测：

1. 先只连开发板并上传程序
2. 再接 OLED，确认能显示 `Booting...`
3. 再接 `DS18B20`
4. 再接 `DHT11`
5. 用手机给 ESP8266 配好 Wi-Fi 和 MQTT
6. 再启动 MQTT，看主题有没有数据
7. 最后测试远程开关命令

## 6. 手机配网怎么做

首次上电，或者设备连不上之前保存的 Wi-Fi 时，ESP8266 会自动开启一个热点。

热点名称类似：

```text
ESP8266-Config-1A2B3C
```

热点密码默认是：

```text
12345678
```

用手机操作：

1. 手机连接这个热点
2. 浏览器一般会自动弹出配网页面
3. 如果没有自动弹出，手动打开：

```text
http://192.168.4.1
```

4. 在页面里填写：
- 家里 Wi-Fi 名称和密码
- `设备名`
- `MQTT服务器地址`
- `MQTT端口`
- `MQTT用户名`
- `MQTT密码`
- `温度校准(摄氏度)`，例如 `0.5` 或 `-0.3`
- `湿度校准(百分比)`，例如 `3.0` 或 `-2.0`
- `时区小时(例如8.0)`，中国可填 `8.0`
- `OTA下载端口`，默认 `1880`
- `OTA固件路径`，默认 `/firmware/esp8266-home-env-control.bin`
- `夜间开始小时(0-23)`，默认 `23`
- `夜间结束小时(0-23)`，默认 `7`
- `允许远程恢复出厂(0禁用/1启用)`，默认 `0`
5. 点击保存
6. 设备会尝试连接家里 Wi-Fi，并把 MQTT 参数保存到 Flash

如果你想恢复出厂：

1. 进入这个配网页面
2. 在 `恢复出厂确认(输入RESET)` 一栏输入：

```text
RESET
```

3. 保存后，设备会清除 Wi-Fi 和 MQTT 配置并自动重启

以后断电重启：

- Wi-Fi 配置不会丢
- MQTT 参数不会丢
- 设备会自动重连

如果只是“当前这一瞬间的传感器数值”，断电后会重新采集；但 MQTT 最新一条上报由于用了 retained，Broker 还能保留最后一次值。

## 7. 串口监视器怎么看

打开 `Tools -> Serial Monitor`，波特率设成：

```text
115200
```

开发板启动后，你应该能看到类似输出：

```text
Device: EnvMonitor-ABC123 | IP: 192.168.1.25 | RSSI: -56 | DS18B20: 24.62 C | DHT11 Humi: 58.00 % | Switch: OFF | Time: 14:08:33 | Error: OK
```

如果设备进入配网模式，串口会看到类似：

```text
进入配网模式
热点名称: ESP8266-Config-1A2B3C
配网页面: 192.168.4.1
```

## 8. MQTT 快速测试

程序正常时，会发布这些主题：

- `home/room1/env01/status`
- `home/room1/env01/temperature`
- `home/room1/env01/humidity`
- `home/room1/env01/switch/state`
- `home/room1/env01/device/info`
- `home/room1/env01/system/state`
- `home/room1/env01/system/event`
- `home/room1/env01/ota/state`

要测试远程控制，可以发一条：

- topic: `home/room1/env01/switch/set`
- payload: `ON`

再发一条：

- topic: `home/room1/env01/switch/set`
- payload: `OFF`

如果你要远程重启，可以用：

- topic: `home/room1/env01/system/reset`
- payload: `RESTART`

如果你确实要远程恢复出厂（默认是禁用），要走双重确认：

- topic: `home/room1/env01/system/reset`
- payload: `ARM_RESET`

然后在 60 秒内再发：

- topic: `home/room1/env01/system/reset`
- payload: `RESET:CONFIRM`

如果你要触发 HTTP OTA 在线升级，可以发：

- topic: `home/room1/env01/ota/set`
- payload: `UPDATE`

如果你想临时指定别的固件路径，也可以发：

- topic: `home/room1/env01/ota/set`
- payload: `UPDATE:/firmware/esp8266-home-env-control-v1.0.0.bin`

## 8.1 OTA 在线升级怎么放固件

项目现在已经把 `Node-RED` 配成了静态文件服务，你把编译好的固件放到这里：

- [docker/nodered/data/public/firmware](../docker/nodered/data/public/firmware)

推荐文件名：

```text
esp8266-home-env-control.bin
```

默认访问地址就是：

```text
http://你的设备IP:1880/firmware/esp8266-home-env-control.bin
```

只要你的配网页里保持默认：

- `OTA下载端口 = 1880`
- `OTA固件路径 = /firmware/esp8266-home-env-control.bin`

那网页按钮或 MQTT 发 `UPDATE` 就会直接从这个地址升级。

如果你要用新版“发布版本号 + 发布说明 + 网页上传”流程，建议用：

- `Node-RED Dashboard` 页面上传 `.bin`
- 自动生成路径：`/firmware/releases/<版本号>/<文件名>.bin`

这样设备升级时你可以发：

- topic: `home/room1/env01/ota/set`
- payload: `UPDATE:/firmware/releases/v1.0.1/esp8266-home-env-control-v1.0.1.bin`

## 9. 如果哪里不对

最常见的排查点：

- OLED 黑屏：`I2C` 模式不对，或者地址不是当前代码里的值
- DS18B20 数值异常：大概率少了 `4.7k` 上拉电阻
- DHT11 显示 `nan`：接线有误，或者读取太频繁
- 时间一直显示 `Time syncing`：先确认 Wi-Fi 已连接，等几十秒看 NTP 是否同步成功
- 配网页面打不开：先确认手机已经连上 ESP8266 热点，再访问 `192.168.4.1`
- Wi-Fi 连不上：家里 Wi-Fi 名称或密码填错
- MQTT 连不上：Broker 地址、端口、用户名、密码有问题
