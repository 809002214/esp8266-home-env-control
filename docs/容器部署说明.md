# Node-RED Docker 部署

这套部署会同时启动：

- `Mosquitto` MQTT 服务
- `Node-RED` 编辑器
- `FlowFuse Dashboard` 网页面板

## 文件位置

- Compose 文件：[docker-compose.yml](../docker/docker-compose.yml)
- MQTT 配置：[mosquitto.conf](../docker/mqtt/config/mosquitto.conf)
- Node-RED 启动脚本：[start.sh](../docker/nodered/data/start.sh)
- Node-RED 初始流程模板：[flows.template.json](../docker/nodered/data/flows.template.json)
- Node-RED 设置文件：[settings.js](../docker/nodered/data/settings.js)

## 首次准备目录和权限

这份 compose 现在是按 `斐讯 N1 / Linux` 的目录结构写的，宿主机路径默认是：

- `/sddata/docker/mqtt/config`
- `/sddata/docker/mqtt/data`
- `/sddata/docker/mqtt/log`
- `/sddata/docker/nodered/data`

第一次部署前，建议先执行：

```bash
mkdir -p /sddata/docker/mqtt/config
mkdir -p /sddata/docker/mqtt/data
mkdir -p /sddata/docker/mqtt/log
mkdir -p /sddata/docker/nodered/data
chmod -R 755 /sddata/docker/mqtt
chmod -R 755 /sddata/docker/nodered
```

如果 `Node-RED` 报 `/data` 没权限，可以继续执行：

```bash
chown -R 1000:1000 /sddata/docker/nodered/data
chmod -R 755 /sddata/docker/nodered/data
```

如果你现在使用的是项目里默认的高权限启动方式：

- `user: "0:0"`
- `privileged: true`

那 `Node-RED` 对目录权限通常会更宽松，但把宿主机目录先建好、权限先设好，还是最稳。

## 先配置 MQTT 账号密码

现在 Mosquitto 已经开启 `用户名密码认证`。

先在 `docker` 目录执行：

### Windows

```powershell
docker run --rm -it -v ${PWD}/mqtt/config:/mosquitto/config eclipse-mosquitto:2 mosquitto_passwd -c /mosquitto/config/passwd your_mqtt_user
```

### 斐讯 N1 / Linux

```bash
docker run --rm -it -v $(pwd)/mqtt/config:/mosquitto/config eclipse-mosquitto:2 mosquitto_passwd -c /mosquitto/config/passwd your_mqtt_user
```

执行后会生成：

```text
docker/mqtt/config/passwd
```

如果你在 N1 上已经使用固定路径，也可以直接执行这条：

```bash
docker run --rm -v /sddata/docker/mqtt/config:/mosquitto/config eclipse-mosquitto:2 mosquitto_passwd -b -c /mosquitto/config/passwd mqtt 123456
```

密码文件创建后，建议把权限顺手修一下：

```bash
chmod 0700 /sddata/docker/mqtt/config/passwd
chown 1883:1883 /sddata/docker/mqtt/config/passwd
```

如果你的系统里没有 `1883:1883` 这个属主，也可以先只执行：

```bash
chmod 0700 /sddata/docker/mqtt/config/passwd
```

这个项目现在已经默认写成：

```yaml
MQTT_USERNAME: mqtt
MQTT_PASSWORD: 123456
```

注意：

- 第一次启动 `Node-RED` 时，会根据这些环境变量自动生成 `flows.json`
- 所以最好在第一次 `docker compose up -d` 之前，就把用户名密码改好

## 启动方法

```bash
cd /你的项目路径/docker
docker compose up -d
```

这份 compose 现在就是按 `斐讯 N1 / Linux` 环境整理的。

第一次启动时，`Node-RED` 会自动安装 `@flowfuse/node-red-dashboard`，通常要多等几十秒到 1 分钟。

## 常用查看命令

```bash
docker compose ps
docker compose logs -f
```

## 访问地址

- Node-RED 编辑器：`http://你的设备IP:1880`
- Dashboard 页面：`http://你的设备IP:1880/dashboard/`

如果是斐讯 N1，就把 `你的设备IP` 换成 N1 的局域网地址，例如：

```text
http://192.168.1.20:1880
http://192.168.1.20:1880/dashboard/
```

## ESP8266 里 MQTT 怎么填

ESP8266 手机配网页里，MQTT 参数这样填：

- `MQTT服务器地址`：运行 Docker 的设备局域网 IP
- `MQTT端口`：`1883`
- `MQTT用户名`：你刚创建的用户名
- `MQTT密码`：你刚创建的密码

例如：

```text
MQTT服务器地址: 192.168.1.20
MQTT端口: 1883
MQTT用户名: mqtt
MQTT密码: 123456
```

## 如果启动失败，先看这几个点

- `1880` 端口是否被占用
- `1883` 端口是否被占用
- 第一次启动时是否给足了安装 Dashboard 节点的时间
- `docker/mqtt/config/passwd` 是否已经生成
- compose 文件里的 `MQTT_USERNAME` 和 `MQTT_PASSWORD` 是否和 `passwd` 里一致
- 斐讯 N1 是否已经装好 `64 位 Armbian / Debian`
- 如果 `Mosquitto` 提示 `Unable to open pwfile`，先检查 `/sddata/docker/mqtt/config/passwd` 是否存在
- 如果 `Mosquitto` 提示 `world readable permissions`，执行 `chmod 0700 /sddata/docker/mqtt/config/passwd`
- 如果 `Node-RED` 提示 `/data/node_modules` 没权限，执行 `chown -R 1000:1000 /sddata/docker/nodered/data`
- 如果 `Node-RED` 页面能打开但没有数据，先看 `docker compose logs -f nodered` 里有没有 `SyntaxError` 或 MQTT 连接失败
