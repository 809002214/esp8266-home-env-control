# Node-RED 网页面板说明

这个项目现在默认使用：

- `Node-RED` 作为流程引擎
- `FlowFuse Dashboard` 作为网页面板

页面地址是：

```text
http://你的设备IP:1880/dashboard/
```

## 页面里已经有哪些内容

### 实时环境

- `温度`
- `湿度`
- `在线状态`
- `开关状态`

### 设备状态

- `本地时间`
- `WiFi 信号`
- `运行时间`
- `设备 IP`
- `最近错误`
- `固件版本`

### 远程控制

- `打开开关`
- `关闭开关`
- `重启设备`
- `恢复出厂`

## 这些数据从哪里来

页面直接订阅你固件上报的 MQTT 主题：

- `home/room1/env01/temperature`
- `home/room1/env01/humidity`
- `home/room1/env01/status`
- `home/room1/env01/switch/state`
- `home/room1/env01/system/state`
- `home/room1/env01/device/info`

控制按钮会往这些主题发送命令：

- `home/room1/env01/switch/set`
- `home/room1/env01/system/reset`

## 页面流程文件在哪里

- 初始模板：[flows.template.json](../docker/nodered/data/flows.template.json)
- 首次启动后生成的实际流程：`docker/nodered/data/flows.json`

## 如果你想继续改页面

最简单的方式：

1. 打开 `Node-RED 编辑器`
2. 在左侧拖拽新的 `ui-text`、`ui-button` 等控件
3. 点右上角 `Deploy`

建议你先不要改 MQTT 主题名，先把这一套跑通。

## 一个很实用的提醒

如果你修改了 compose 里的 MQTT 用户名密码，而 `Node-RED` 已经生成过 `flows.json`，那么旧流程里的 broker 配置不会自动跟着变。

这时有两种处理方式：

1. 直接在 `Node-RED 编辑器` 里修改 MQTT Broker 配置并重新部署
2. 删除 `docker/nodered/data/flows.json` 后重启容器，让模板重新生成

## 如果页面能打开但看不到数据

优先检查这 3 个地方：

1. `Node-RED` 日志里有没有 `Connected to broker`
2. `Node-RED` 日志里有没有 `SyntaxError`
3. `ESP8266` 是否已经在线并持续上报这些主题

常用命令：

```bash
cd /sddata/docker
docker compose logs -f nodered
docker compose logs -f mqtt
```
