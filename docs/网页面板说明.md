# Node-RED 网页面板说明

这个项目现在默认使用：

- `Node-RED` 作为流程引擎
- `FlowFuse Dashboard` 作为网页面板

页面地址是：

```text
http://你的设备IP:1880/dashboard/
```

Node-RED 编辑器地址和默认登录信息：

- 编辑器：`http://你的设备IP:1880`
- 用户名：`admin`
- 密码：`123456`

## 页面里已经有哪些内容

### 欢迎区

- 顶部欢迎横幅
- 当前页面用途说明

### 实时环境卡片

- `温度`
- `湿度`
- `在线状态`
- `开关状态`

### 历史曲线

- `最近 12 小时温度曲线`

- `重启原因`

说明：远程恢复出厂默认禁用，需要先在设备配网页把 `允许远程恢复出厂` 从 `0` 改为 `1`。

## 这些数据从哪里来

页面直接订阅你固件上报的 MQTT 主题：

- `home/room1/env01/temperature`
- `home/room1/env01/humidity`
- `home/room1/env01/status`
- `home/room1/env01/switch/state`
- `home/room1/env01/system/state`
- `home/room1/env01/device/info`
- `home/room1/env01/ota/state`

页面模板里还内置了一条天气发布流（Node-RED 主动发布）：

- `home/room1/env01/weather/state`

控制按钮会往这些主题发送命令：

- `home/room1/env01/switch/set`
- `home/room1/env01/system/reset`
- `home/room1/env01/ota/set`（发送 `UPDATE:/firmware/releases/<版本号>/<文件名>.bin`）

## 天气流（已内置）

默认模板已经包含这条链路：

`定时触发(15分钟)` -> `Open-Meteo API` -> `映射天气` -> `发布 MQTT`

发布主题：

- `home/room1/env01/weather/state`

默认坐标是成都（`30.5728, 104.0668`）。如果你要改城市：

1. 打开 Node-RED 编辑器
2. 找到节点 `构建天气请求URL`
3. 修改 `lat` / `lon` 默认值
4. 点击 `Deploy`

## 历史曲线现在是什么方式

当前这版历史曲线是 `Node-RED Dashboard` 自己在运行时内存里保存最近数据，默认展示：

- `最近 12 小时`

这意味着：

- 页面刷新不会丢
- `Node-RED` 容器重启后，曲线会重新开始累计

如果你后面要做：

- 跨天历史
- 7 天 / 30 天曲线
- 历史导出

那下一步就该加数据库，比如 `InfluxDB` 或 `SQLite`。

## 页面流程文件在哪里

- 初始模板：[flows.template.json](../docker/nodered/data/flows.template.json)
- 首次启动后生成的实际流程：`docker/nodered/data/flows.json`

## 规范 OTA 发布流程

推荐用这一套固定流程：

1. 在 Dashboard 填 `发布版本号` 和 `发布说明`
2. 上传 `.bin` 固件
3. 在下拉框里选中目标版本
4. 点击 `升级所选固件`

上传成功后，系统会自动写入：

- 固件文件：`docker/nodered/data/public/firmware/releases/<版本号>/<文件名>.bin`
- 发布说明：`docker/nodered/data/public/firmware/releases/<版本号>/release.md`
- 固件清单：`docker/nodered/data/public/firmware/catalog.json`

设备收到的 OTA 命令格式是：

```text
UPDATE:/firmware/releases/<版本号>/<文件名>.bin
```

## 如果你想继续改页面

最简单的方式：

1. 打开 `Node-RED 编辑器`
2. 在左侧拖拽新的 `ui-text`、`ui-button` 等控件
3. 点右上角 `Deploy`

建议你先不要改 MQTT 主题名，先把这一套跑通。

## 一个很实用的提醒

如果你修改了 compose 里的 MQTT 用户名密码，或者项目里升级了默认页面模板，而 `Node-RED` 已经生成过 `flows.json`，那么旧流程不会自动跟着变。

这时有两种处理方式：

1. 直接在 `Node-RED 编辑器` 里修改 MQTT Broker 配置并重新部署
2. 删除 `docker/nodered/data/flows.json` 后重启容器，让模板重新生成

常用命令：

```bash
cd /sddata/docker
rm -f /sddata/docker/nodered/data/flows.json
docker compose restart nodered
```

## 如果页面能打开但看不到数据

优先检查这 3 个地方：

1. `Node-RED` 日志里有没有 `Connected to broker`
2. `Node-RED` 日志里有没有 `SyntaxError`
3. `ESP8266` 是否已经在线并持续上报这些主题

常用命令：

```bash
cd /sddata/docker
docker compose logs -f nodered
docker compose logs -f mqtt
```
