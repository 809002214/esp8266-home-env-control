# 接线说明

这份说明已经按你实际买到的模块重新核对过：

- `NodeMCU V3 (ESP8266)`
- `DS18B20` 接 `D5 / GPIO14`
- `DHT11` 接 `D6 / GPIO12`
- `OLED 7 针屏` 按 `SPI` 模式接
- 远程开关控制预留 `D7 / GPIO13`

## 接线前先看这两点

1. `ESP8266` 是 `3.3V` 逻辑，传感器和 OLED 优先都接 `3V3`。
2. 你发来的 OLED 商品图已经明确写了排针定义：`CS / DC / RES / D1 / D0 / VCC / GND`，所以它是 `SPI` 接法，不是 `I2C`。

## 主接线表

| 模块 | 引脚 | 接到哪里 | 说明 |
|---|---|---|---|
| DS18B20 | VCC | `3V3` | 推荐 3.3V |
| DS18B20 | GND | `GND` | 共地 |
| DS18B20 | DATA | `D5` | `GPIO14` |
| DHT11 模块 | VCC | `3V3` | 推荐 3.3V |
| DHT11 模块 | GND | `GND` | 共地 |
| DHT11 模块 | DATA | `D6` | `GPIO12` |
| OLED(SPI) | GND | `GND` | 共地 |
| OLED(SPI) | VCC | `3V3` | 推荐 3.3V |
| OLED(SPI) | D0 | `D1` | `GPIO5`，SPI 时钟 |
| OLED(SPI) | D1 | `D2` | `GPIO4`，SPI 数据 |
| OLED(SPI) | RES | `D0` | `GPIO16` |
| OLED(SPI) | DC | `D3` | `GPIO0` |
| OLED(SPI) | CS | `D4` | `GPIO2` |
| 后续继电器模块 | IN | `D7` | `GPIO13` |

## DS18B20 详细接法

`DS18B20` 的数据线需要一个上拉电阻。

按下面接：

- `DS18B20 VCC -> 3V3`
- `DS18B20 GND -> GND`
- `DS18B20 DATA -> D5`
- `DATA` 和 `3V3` 之间加一个 `4.7k` 电阻

示意图：

```text
3V3 ----+---------------------> DS18B20 VCC
        |
        +--[4.7k]--+
                   |
D5 ----------------+---------> DS18B20 DATA
GND -------------------------> DS18B20 GND
```

你这次发来的图片是 `TO-92` 裸管版本，所以这个电阻要你自己加。

并且注意脚位方向：

- 平面朝自己、引脚朝下
- 左边 `GND`
- 中间 `DQ`
- 右边 `VDD`

## DHT11 详细接法

大多数 `DHT11` 模块都是 3 针：

- `VCC`
- `GND`
- `DATA`

接法如下：

```text
3V3 -------------------------> DHT11 VCC
D6 --------------------------> DHT11 DATA
GND -------------------------> DHT11 GND
```

如果你手里的是 4 针裸 `DHT11`，先别急着接，最好先确认一下引脚定义。

## OLED 详细接法

你这块 OLED 不是 `I2C`，而是 `SPI`。

按下面接：

```text
GND -------------------------> OLED GND
3V3 -------------------------> OLED VCC
D1 / GPIO5 ------------------> OLED D0   (SPI时钟)
D2 / GPIO4 ------------------> OLED D1   (SPI数据)
D0 / GPIO16 -----------------> OLED RES
D3 / GPIO0 ------------------> OLED DC
D4 / GPIO2 ------------------> OLED CS
```

几个关键提醒：

- `D3` 和 `D4` 是启动相关脚，接线一定不要短路
- 这块屏接对线以后，代码也必须改成 `SPI SSD1306` 驱动方式
- 如果还是用之前的 `I2C` 代码，屏幕大概率会黑

## 远程开关预留脚位

示例代码里已经把 `D7` 预留给开关控制。

现在你可以先不接任何东西。

以后如果加继电器模块，通常这样接：

- `Relay IN -> D7`
- `Relay VCC -> 模块要求的电压`
- `Relay GND -> GND`

不要把灯、风扇或其它负载直接接到 `ESP8266` 引脚上。

## 一定要共地

下面这些地线必须连在一起：

- `NodeMCU GND`
- `DS18B20 GND`
- `DHT11 GND`
- `OLED GND`
- 后续继电器模块 `GND`

不共地的话，经常会出现读数乱跳、设备没反应、控制异常。

## 推荐实际动手顺序

建议按这个顺序逐步接线和测试：

1. 先只接 `OLED`
2. 再接 `DS18B20`
3. 再接 `DHT11`
4. 再连 Wi-Fi 和 MQTT
5. 最后再用 `D7` 做继电器或 LED 控制测试

## 目前这版接线的最终结论

按你发来的 4 个模块，当前最合适的接法就是：

- `DS18B20 DQ -> D5`
- `DHT11 DATA -> D6`
- `OLED D0 -> D1`
- `OLED D1 -> D2`
- `OLED RES -> D0`
- `OLED DC -> D3`
- `OLED CS -> D4`
- `继电器 IN -> D7`
