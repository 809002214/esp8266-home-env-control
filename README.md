# ESP8266 家庭环境监测与远程控制系统

基于 `ESP8266`、`MQTT`、`Node-RED` 和 `OLED` 的家庭温湿度监测与远程控制入门项目。

## 仓库信息

- GitHub 仓库名推荐：`esp8266-home-env-control`
- 项目描述推荐：`基于 ESP8266、MQTT、Node-RED 和 OLED 的家庭温湿度监测与远程控制入门项目。`

这是一个适合新手的家庭环境监测与远程控制项目，目标是：

- `ESP8266 NodeMCU` 连接家里 Wi-Fi
- 采集温度、湿度
- 通过 `MQTT` 上报数据
- 在 `OLED` 上本地显示
- 在网页上实时查看数据
- 预留远程开关控制功能

## 一句话结论

你现在这套器件可以做出一个完整的入门系统，但有两个现实注意点：

- 你已经有 `DS18B20` 和 `DHT11`，其中 `DS18B20` 负责温度，`DHT11` 负责温度+湿度，所以温度会重复。实际项目里建议：
  - 用 `DS18B20` 作为“更稳定的温度”
  - 用 `DHT11` 只取“湿度”，同时把它的温度当作参考值
- 你的 `0.96 寸 OLED 7 针` 很可能是 `I2C/SPI 双模式` 屏，接线前一定先看模块丝印。很多 7 针屏默认不是纯 I2C 接法。

## 推荐的整体架构

最适合新手的落地路线：

`ESP8266` -> `家里 Wi-Fi` -> `MQTT Broker` -> `网页面板`

建议优先选下面这个组合：

- 设备端：`ESP8266 + Arduino`
- MQTT 服务：`Mosquitto` 或 `EMQX`
- 网页面板：`Node-RED + FlowFuse Dashboard`

原因：

- 上手门槛最低
- 网页、卡片、按钮、历史图表都现成
- 后面加继电器、门磁、人体传感器也方便
- 不需要你一开始就写服务器和前端

如果你准备把服务长期跑在 `斐讯 N1` 上，建议先看一遍 [容器部署说明.md](./docs/容器部署说明.md)，里面已经补了：

- `/sddata/docker` 目录准备命令
- `mqtt/config/passwd` 的创建方式
- `chmod` / `chown` 权限修复命令
- `Node-RED` 和 `Mosquitto` 常见报错排查

## 网页方案怎么选

### 方案 A：Node-RED

当前更推荐。

适合你如果：

- 设备存储比较小，比如 `斐讯 N1 8G`
- 想快速做一个实时网页和控制面板
- 喜欢拖拽流程，后面还想加自动化逻辑

你能直接得到：

- 实时温湿度显示
- 开关按钮
- 设备状态页
- 手机网页可访问
- 后期自动化联动

### 方案 B：Home Assistant

功能更完整，但对存储空间要求更高。

适合你如果：

- 有更大的存储空间
- 想直接要历史、设备页、自动发现、更多家庭自动化能力
- 不介意资源占用更高

注意：如果你只是想在 `斐讯 N1 8G` 上长期稳定跑一个温湿度网页，`Node-RED` 会更轻量。

### 方案 C：自己写网页

可以，但不建议你作为第一步就做。

典型做法是：

- 浏览器通过 `WebSocket` 连 MQTT Broker
- 或者网页请求你自己写的后端，再由后端订阅 MQTT

这条路更灵活，但你需要同时处理：

- 前端页面
- 实时通信
- 鉴权
- 数据存储
- 开关控制状态同步

对新手来说会明显更绕。

## 你这套硬件怎么分工

### 1. NodeMCU V3

主控，负责：

- 连 Wi-Fi
- 读取传感器
- 驱动 OLED
- 发布/订阅 MQTT

### 2. DS18B20

数字温度传感器，优点：

- 温度比 `DHT11` 更靠谱
- 适合长期测温

建议用途：

- 作为主温度值上报到网页

### 3. DHT11

温湿度传感器，优点：

- 简单便宜

缺点：

- 精度一般
- 速度慢
- 温度部分通常不如 `DS18B20`

建议用途：

- 用来读湿度
- 温度值只做参考

### 4. OLED 0.96 128x64

用来本地显示：

- 当前温度
- 当前湿度
- Wi-Fi 状态
- MQTT 状态
- 开关状态

## 新手推荐接线

下面先按“`OLED 使用 I2C 模式`”给你一版最容易接的方案。

### 建议引脚分配

- `DS18B20 DATA` -> `D5` (`GPIO14`)
- `DHT11 DATA` -> `D6` (`GPIO12`)
- `OLED SDA` -> `D2` (`GPIO4`)
- `OLED SCL` -> `D1` (`GPIO5`)
- 预留继电器控制 -> `D7` (`GPIO13`)

### 电源接法

- 所有模块 `GND` 都接一起
- 传感器和 OLED 优先接 `3V3`

原因：

- `ESP8266` 是 `3.3V` 逻辑
- 直接统一 `3.3V` 更安全

### DS18B20 接线

标准三线：

- `VCC` -> `3V3`
- `GND` -> `GND`
- `DATA` -> `D5`

并且要注意：

- `DATA` 和 `3V3` 之间通常需要一个 `4.7k` 上拉电阻
- 如果你买的是成品模块，有些板子已经自带上拉电阻

### DHT11 模块接线

如果你买的是三针模块，通常是：

- `VCC` -> `3V3`
- `GND` -> `GND`
- `DATA` -> `D6`

如果是四针裸传感器，接法不同，不建议新手优先用裸传感器版本。

### OLED 接线

如果你的 OLED 目前是 `I2C` 模式，常见是：

- `VCC` -> `3V3`
- `GND` -> `GND`
- `SDA` -> `D2`
- `SCL` -> `D1`

但你这个是 `7 针` 屏，要特别注意：

- 有些 7 针屏支持 `4 线 SPI`
- 有些支持 `I2C/SPI`，靠电阻位或焊盘切换
- 丝印可能写成 `D0/D1/RES/DC/CS` 之类，而不是 `SDA/SCL`

如果你发现屏上没有明确 `SDA/SCL`，先不要硬接，我建议你拍一下 OLED 正反面丝印再确认。

## MQTT 主题建议

建议从一开始就把主题命名规范好。

设备名先定成：

- `home/room1/env01`

建议这些主题：

- 上报温度：`home/room1/env01/temperature`
- 上报湿度：`home/room1/env01/humidity`
- 上报在线状态：`home/room1/env01/status`
- 上报开关状态：`home/room1/env01/switch/state`
- 接收开关命令：`home/room1/env01/switch/set`

消息例子：

- 温度：`24.8`
- 湿度：`58`
- 在线状态：`online` / `offline`
- 开关命令：`ON` / `OFF`

## OLED 页面建议

第一页显示：

- `Temp: 24.8 C`
- `Humi: 58 %`
- `WiFi: OK`
- `MQTT: OK`

第二页显示：

- `Switch: ON/OFF`
- `IP: 192.168.1.xxx`
- `RSSI: -55 dBm`

## 远程开关控制怎么做

控制链路大概是：

网页按钮 -> MQTT 发布命令 -> ESP8266 订阅命令 -> 控制 GPIO -> 回报状态

也就是说：

1. 你在网页上点“开”
2. 平台往 `home/room1/env01/switch/set` 发 `ON`
3. ESP8266 收到后把 `D7` 拉高或拉低
4. 如果接了继电器，就会真的控制外部设备
5. ESP8266 再往 `switch/state` 回报当前状态

这是标准做法，后面扩展特别方便。

## 新手非常重要的几个提醒

### 1. 不要直接用 GPIO 去带大功率设备

`ESP8266` 的 IO 口只能输出很小的电流。

如果你以后要控制：

- 灯
- 风扇
- 插座

应该使用：

- `继电器模块`
- 或 `MOS 管模块`

不要把负载直接接到开发板引脚上。

### 2. 先别碰市电 220V

如果你是纯新手，第一阶段只做：

- 传感器
- OLED
- MQTT
- 一个低压 LED 或小模块做“开关演示”

等你完全熟悉后，再考虑继电器控制市电设备。

### 3. 启动脚要避坑

`ESP8266` 某些引脚会影响启动模式，所以不建议乱选引脚做外设。上面给你的 `D1/D2/D5/D6/D7` 是比较常见、相对稳妥的选择。

### 4. DHT11 不要采样太快

建议每 `2~5 秒` 读一次就可以了，不要高频读取。

### 5. MQTT 要做断线重连

项目真正稳定与否，不是看“能不能跑起来”，而是看：

- Wi-Fi 掉线后能不能自动恢复
- MQTT 断开后能不能自动重连

## 推荐实施顺序

按这个顺序做，成功率最高：

1. 先只点亮 `OLED`
2. 再单独读取 `DS18B20`
3. 再单独读取 `DHT11`
4. 把两组数据显示到 OLED
5. 再接入 Wi-Fi
6. 再接入 MQTT
7. 再做网页显示
8. 最后再接“远程开关”

不要一上来把所有模块一起接上调，会很容易卡住。

## 我给你的实际建议

如果你想少走弯路，我建议你这样做：

- 现在先用 `Node-RED + Mosquitto`
- ESP8266 先完成“采集 + OLED + MQTT 上报”
- 网页端先不自己写，先用 `Node-RED Dashboard` 做可视化
- 远程控制先做“虚拟开关”
- 确认逻辑通了，再接继电器
